Qué necesitará para configurar un servidor de correo electrónico personal

  • -Una computadora separada con suficiente capacidad de disco duro, que actuará como servidor de correo electrónico.
  • -Nombre de dominio para el servidor de correo electrónico que usará para configurar direcciones de correo electrónico.
  • -Conexión a Internet confiable y de alta velocidad.
  • -Un sistema operativo como Windows o Linux para ejecutar el servidor.
  • -Programas de software como MailEnable que pueden ordenar y enrutar correos electrónicos, protección antivirus y filtro de spam como SpamAssassin (el software que usará depende del sistema operativo en el que se establezca).
  • -Un sistema para agilizar el proceso de configuración.

¿Qué es hMailServer?

es un servidor de correo electrónico gratuito para organizaciones, empresas e individuos que es fácil de instalar y fácil de usar. Descubra cómo instalar y configurar hMailServer en este resumen.

    ¿Es hMailServer seguro?

    La seguridad de hMailServer depende de los pasos que tome durante su configuración, instalación y configuración inicial. Fuera de la caja, no será seguro. No se producirá ninguna transferencia segura de correo electrónico.

    Los puertos predeterminados como 110 ( POP3 ), 25 ( SMTP ), 143 ( IMAP ) no son seguros. Sin embargo, cuando agrega un certificado SSL y crea nuevos enlaces en el software, obtendrá una transferencia segura de correo electrónico a través de los puertos 995 (POP3 SSL), 465 (SMTP SSL) y 993 (IMAP SSL).

    Le recomiendo que compre un certificado SSL de terceros para el nombre de host de su nuevo hMailServer antes de enviar o recibir correos electrónicos, especialmente para una cuenta de correo electrónico comercial profesional. Dejar un servidor de correo electrónico POP3/SMTP inseguro en Internet no es lo más inteligente en estos días.

    ¿Qué significa SMTP?

    El Simple Mail Transfer Protocol (SMTP) se utiliza para enviar correos desde un cliente de correo a un email server o hosting de email. También se usa para reenviar o transmitir correos de un email server a otro. La habilidad de transmitir correos de un servidor a otro es necesaria si el remitente y el receptor tienen proveedores de correo diferentes.

    ¿Qué significa IMAP?

    Internet Message Access Protocol (IMAP), es el protocolo más sofisticado actualmente. Les permite a los usuarios agrupar todos los correos relacionados y archivarlos en carpetas, que luego pueden ser organizados de forma jerárquica. Tiene marcadores en los mensajes que informan cuando un correo ha sido leído, eliminado o reenviado. Además de que les permite a los usuarios realizar búsquedas en su buzón.

    ¿Qué significa POP?

    El Post Office Protocol, es usado para recuperar mensajes de correo electrónico de un servidor de correo a un cliente de correo. POP versión 3 (versión más reciente), soporta extensiones y una gran cantidad de mecanismos de autenticación. Estas características son necesarias para prevenir que los usuarios maliciosos se hagan con el permiso para acceder a los mensajes de los usuarios.

Configuración DNS

Como mencioné anteriormente, hay algunos registros DNS que serán necesarios para que funcione la funcionalidad completa del software. Primero, manejemos registros externos (en Internet). El primero es un registro MX .

Un registro MX DNS es un registro que indica cómo/dónde enviar un correo electrónico a un dominio específico. Este registro debe apuntar a la dirección [IP] 'externa' de su hMailServer. Como ejemplo, un correo electrónico destinado a @reindersconsulting.com podría pasar por hmail.reindersconsulting.com.

A continuación, necesitaría un registro A externo . Esto le dice a los servidores DNS qué dirección IP está asociada con hmail.reindersconsulting.com.

Sus administradores de red podrían asignar una dirección IP externa para su hMailServer. Un escenario común es tener una configuración de NAT 'uno a uno' en su firewall perimetral. Esto permite que el tráfico entrante a la dirección IP externa de su hMailServer se 'traduzca' a su dirección IP interna (LAN) (por ejemplo, 10.0.1.154).

Ahora, para cubrir los registros internos, el escenario más simple es un registro A para el nombre de host de su hMailServer que apunta a su dirección IP interna (LAN). Podría tener 'hmailserver01.reinders.local' apuntando a 10.0.1.154.

Esta es la configuración más simple absoluta. Puede realizar investigaciones adicionales sobre configuraciones de DNS más sofisticadas.

Puertos de cortafuegos/enrutador

Como insinué anteriormente, necesitará algunos puertos abiertos en su enrutador de borde o firewall para permitir que el tráfico saliente fluya entre su hMailServer e Internet. Supongamos que está utilizando solo tráfico seguro (SSL). En ese caso, puede pedirle a su equipo de red que abra los siguientes puertos entrantes desde Internet a la dirección IP local (LAN) de su hMailServer.

  • -465 (SMTP seguro)
  • -995 (POP3 seguro, si es necesario)
  • -993 (IMAP seguro)

    • Especificación de un nombre de host público y un repetidor SMTP

    Para permitir el tráfico de correo electrónico entrante desde Internet, debe especificar un nombre de host público de la computadora en la que instaló hMailServer. Lo hace a través de Configuración -> Protocolos -> SMTP. A continuación, haga clic en la pestaña Entrega de correo electrónico.

    En el campo 'Nombre de host local', podría ingresar 'hmailserver01.reindersconsulting.com. Ahora, para que eso funcione, necesitaría asegurarme de tener un registro DNS A externo (Internet) para que ese nombre de host apunte a la dirección IP externa de mi servidor.

    En términos de correo electrónico saliente, es típico que las redes de la empresa bloqueen el puerto 25 saliente directamente desde un servidor a Internet. Esto se hace por motivos de seguridad. Sin embargo, puede verificar si su servidor tiene la capacidad de enviar en el puerto 25 o no.

    Vaya a un símbolo del sistema/Terminal e ingrese este comando:

    correo telnet servidor de correo electrónico com 25

    Cifrado de correos electrónicos con certificación SSL

    Como ha notado hasta ahora, la instalación y configuración inicial del servidor no incluye el tráfico de correo electrónico seguro. Para poder configurar su nuevo servidor de correo electrónico, necesitará un certificado SSL para la autenticación y el cifrado de una autoridad de certificación (CA) de confianza como DigiCert , SSLS.com o CheapSSLSecurity.com , por nombrar algunos. Puede seguir las instrucciones del proveedor de certificados SSL para obtener su certificado. Asegúrese de obtener el certificado SSL en formato PEM , compatible con hMailServer. Necesitará el archivo y los archivos de clave privada disponibles. Para importar su nuevo certificado, vaya a Configuración -> Avanzado -> Certificados SSL. Haga clic en 'Agregar...' Escriba lo que quiere que sea el nombre. Probablemente sea mejor usar el 'Nombre del sujeto' de su certificado. Luego, busque el 'Archivo de certificado' y el 'Archivo de clave privada'. A continuación, haga clic en Guardar. Ahora que se agregó el certificado SSL, debe "vincularlo" a sus puertos seguros. En la sección Avanzado, vaya a 'Puertos TCP/IP'.

    Cifrado de correos electrónicos con certificación SSL

    Debido a que estamos usando puertos seguros, necesitamos agregar uno. Haga clic en 'Agregar' e ingrese la información SMTP según la captura de pantalla.

    El único otro paso que daría, por supuesto, es elegir su nuevo certificado SSL del menú desplegable 'Certificado SSL'. Puede realizar los mismos pasos para POP3 e IMAP usando los mismos campos esenciales, el mismo certificado, pero use los puertos 995 para POP3 y 993 para IMAP. ¡Muy fácil!
